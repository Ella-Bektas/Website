Streamlining Data Upload

for Alchemite Analytics

Creation of a completely new "Add Data" page in Alchemite Analytics, designed to provide users with a quick and effortless means of adding new experiments to their model, thereby improving efficiency and usability.

About

Alchemite Analytics is a machine learning platform, developed to optimise R&D in materials, chemicals and life sciences. This project presents an in-depth exploration of my design journey, focusing on the transformation of the "Add Data" page. This project was a complete redesign, transitioning it from a simple CSV text box to an intuitive, editable table.

Research

Original Add Data Page

After listening to our users, it became clear that they faced challenges when adding data to their existing models. Instead of using the "Add Data" page, they often resorted to creating new models because our "Create Model" page with its drag-and-drop interface was more intuitive.

Image of Original Add Data Page

The Problem

This work around led to difficulties in uploading, editing, and saving data efficiently. As a result, users couldn't fully leverage the benefits of Alchemite's "Model Revisions", which show the Model's improvement with each new data addition. This feature only works when data is added to an existing Model, not when new Models are created.

Purpose

User Goals

After gathering insights from customer requests and conducting user interviews, we discovered a common user need. Individuals who perform new experiments in the lab expressed a strong desire to seamlessly upload this data into their Alchemite Model. Our research allowed us to pinpoint the key objectives users have when adding data:

  • Have the ability to drag and drop excel files onto the page 
    •
  • Add single and multiple rows of data at a time
    •
  • Be able to easily edit and review the data adding before saving it to the Model
Our Goals

The business goal for the year was to improve customer retention to >90%. In turn, our product goal was to increase customer satisfaction and reduce the time to value for users in Alchemite Analytics.

Define

User Workflow

After interviewing our users, we gained a deep understanding of their preferred data addition workflow. Designing to meet these user needs and streamline the process is key to saving time and delivering value by eliminating unnecessary steps.

Feature Requirements

User Objectives
  • Easily be able to upload their data 
    •
  • Upload both single and multiple rows at once
    •
  • Be able to easily edit and review the data adding before saving it to the Models
New Feature Requirements
  • Include a drag and drop tile which is visible on the interface 
    •
  • Compatible with single and batch rows
    •
  • Editable table which can be reviewed for the data is saved to the Model

Ideation

Lo-Fi Wireframes

During the ideation stage, I worked closely with the Product Manager and Engineering team to generate and develop versions of the add data page which would be both valuable to users and feasible to build. We went through rounds of brainstorming and developed storyboards to help us generate design solutions.

Usability Tests

After creating initial designs within the product trio, we subjected the designs to user testing. This feedback highlighted two key challenges with the new page:

  • The design did not clearly indicate that the cells were editable 
    •
  • There was no obvious confirmation mechanism for adding data to the Model
Responding to Feedback

In response to the feedback, we implemented the following design changes:

  • Permanently displaying editable cells, using UI elements like hover effects. 
    •
  • Introducing a confirmation modal that appears when users click the 'Save to Model' button.

First Release

As our engineering team operates with continuous delivery, we were able to launch a first release. The first version was an editable table interface that allowed users to either input data manually in a single row or conveniently drag and drop an entire dataset to auto-populate the table. Users had the flexibility to make edits as needed before saving the data to create a new Model.

Agile Changes and Final Result

After the initial release, we assessed user feedback and observed that the feature was generally more user-friendly than the previous page. Yet, there were a few areas that required enhancement. In response, we adopted an agile approach and promptly made UI adjustments to further enhance the user experience.

Visible Inputs

In our initial release, the input fields were not immediately visible, making it difficult for new users to discern their editability. As a result, we implemented a minor UI adjustment to ensure the input fields remained consistently available, rather than requiring a click to reveal them.

Success Banner

When uploading data, sometimes the data uploaded was not formatted correctly. Users were getting confused. As a solution, we implemented a success and error banner which would change depending of whether or not the data was compatible.

Retraining Modal

In the initial designs, we included a name input box on the interface. However, this proved to be confusing for users as it was constantly visible. To address this issue, we relocated the naming input to a Modal, which is activated when the user presses the create button
